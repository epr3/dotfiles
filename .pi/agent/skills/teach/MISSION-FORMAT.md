# MISSION.md Format

Workspace root. The *reason* the user is learning; every teaching decision traces back here.

```md
# Mission: {Topic}

## Why
{1-3 sentences. The concrete real-world goal — what changes in their life/work with this skill. Not "to understand X"; push for the outcome.}

## Success looks like
- {Specific, observable things the user will be able to do}

## Constraints
- {Time, budget, commitments, learning preferences}

## Out of scope
- {Adjacent topics explicitly not chased now — protects the zone of proximal development}
```

Rules: **one mission per workspace** (two topics = two workspaces) · **concrete over abstract** ("run a half marathon by October" beats "get fitter") · **push back on vagueness** — can't articulate why → interview before writing; a bad mission is worse than none · **revise when reality shifts** — don't let a stale mission steer · **keep it under a screen** — past that it's a plan, not a compass.
