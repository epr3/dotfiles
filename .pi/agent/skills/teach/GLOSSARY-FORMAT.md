# GLOSSARY.md Format

Canonical language for the workspace; all lessons, references, and learning records adhere to it. Building it is itself learning — compressing a concept into a tight definition is evidence of understanding.

```md
# {Topic} Glossary

{One-two sentences on what this covers.}

## Terms

**Progressive overload**:
Systematically increasing the demand on a muscle over time — via load, volume, or intensity.
_Avoid_: Pushing harder, levelling up
```

Rules: **add a term only once the user understands it** — record of compressed knowledge, not a dictionary to learn from · **be opinionated** — pick the best word, list the rest as _Avoid_ aliases · **tight definitions** — one-two sentences, what it IS · **use glossary terms inside definitions** — that's what makes complex terms graspable · **subheadings when clusters emerge**, flat list otherwise · **flag ambiguities** ("here, 'set' always means a working set") · **revise in place** as understanding deepens.
